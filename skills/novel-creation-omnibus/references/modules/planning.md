## 2.3 细纲文件管理

```
📂 目录结构：
  大纲/
  ├── 总大纲.md
  ├── 细纲/
  │   ├── 细纲索引.md
  │   ├── 第1卷/第1章_细纲.md、第2章_细纲.md……
  │   ├── 第2卷/……
  │   └── 废稿备份/

📝 细纲索引（大纲/细纲/细纲索引.md）：
  | 章号 | 文件路径 | 标题 | 状态 | 版本 | 预计字数 | 实际字数 |
  |:-----|:---------|:-----|:-----|:-----|:---------|:---------|

🚨 规则：
  - 每章细纲独立成文件，不合并
  - 写正文前必须先读取对应细纲文件
  - 细纲不存在→提示先创建
  - 修改细纲前自动备份旧版
```

## 2.4 一键全检（9维度）

```
□ 主线检查：当前剧情是否偏离核心主线？
□ 人设检查：人物行为是否符合人设（无OOC）？
□ 节奏检查：爽点密度/钩子密度/情绪曲线是否合理？
□ 伏笔检查：待回收伏笔是否被遗忘？
□ 一致性检查：世界观/设定/人物状态是否自洽？
□ 字数检查：每章是否达标？
□ 衔接检查：章间是否存在断裂/时间跳跃？
□ 平台合规检查：是否符合平台要求？
□ 命名检查：章节命名风格是否统一？
```
