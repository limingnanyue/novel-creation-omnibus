# 第14章：网文生产标准流程（14节点）

> 适用："网文生产""锁稿""审稿""交接"

---

## 14.1 14节点完整流程

```
节点1：初始化/窗口定位 → 确认项目+最高锁稿章+唯一生产源
节点2：入口复核 → 判断承接/禁区/功能/可用素材（不写正文）
节点3：任务卡 → 章名/功能/必须兑现/禁写/出场人物/钩子/字数
节点4：生产前控制卡 → 确认不跑偏
节点5：外部素材接入 → 拆出可用动作对白，删未授权设定
节点6：正式整合清瑕 → 删污染句/换抽象词/压重复/校一致
节点7：连续性审查 → 时间线/位置/知识范围/物件链/规则/章尾
节点8：反方审稿 → P0-P3四级
节点9：定点小修 → 只修审稿问题
节点10：机械复检 → 错别字/标点/格式
节点11：字数审计 → 含中文标点的可见字符数
节点12：锁稿确认 → 版本唯一+审稿已处理+字数清楚
节点13：小结与挂账 → 记录后续必须承接的信息
节点14：交接卡 → 最高锁稿章/生产源/禁区/挂账
```

## 14.2 反方审稿四级

```
P0：安全/平台红线/严重AI痕迹/字数不达标 → 必改
P1：因果断裂/OOC/设定冲突/时间线错/物件链断 → 必改
P2：节奏拖沓/爽点不落地/钩子弱/信息密度失衡 → 建议改
P3：重复词/标点密度/局部句子不顺 → 可改可不改

总裁决：A可锁稿 / B小修后锁 / C返修后复审 / D结构重做
```

## 14.3 锁稿6条件

```
① 正文版本唯一
② 审稿问题已处理或明确挂账
③ 字数统计口径清楚
④ 章节实体结果清楚
⑤ 下一章入口挂账清楚
⑥ 撤回版本已标注不得使用
```
