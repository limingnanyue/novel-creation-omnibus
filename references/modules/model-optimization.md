# 第26章：模型适配优化（Model-Specific Optimization）

> 适用：任何写作任务——自动感知当前模型，选择最优策略
> 场景：无论用户用哪家模型，都能写出"有味道"不AI味的中文网文

---

## 26.1 模型识别与路由

写正文前自动执行以下识别流程：

```
任务开始 → 检查系统提示中当前使用的模型
         ├─ 出现"Claude"/"Sonnet"/"Haiku" → Claude分支
         ├─ 出现"DeepSeek"/"deepseek"       → DeepSeek分支
         ├─ 出现"GPT"/"gpt-4"/"gpt-3.5"     → GPT分支
         ├─ 出现"Qwen"/"通义"/"千问"        → Qwen分支
         ├─ 出现"Kimi"/"Moonshot"           → Kimi分支
         ├─ 出现"Doubao"/"豆包"             → 豆包分支
         └─ 其他 → 通用分支
```

## 26.2 Claude 优化分支

### 特性诊断

| 维度 | 表现 | 严重程度 |
|:-----|:-----|:---------|
| 文学性 | 极强，善用比喻排比 | ⚠️ 优点，但容易过量 |
| 中文水准 | 优秀但偶有翻译腔 | ⚠️ 注意 |
| 禁用词倾向 | "仿佛/似乎/轻叹/嘴角"频率高 | 🔴 高 |
| 道德感 | 有"说教"冲动 | 🔴 高 |
| 情绪描写 | 细腻丰富 | ✅ 优点 |

### 负面提示专属

```
Claude专用深层负面提示（写章节前注入）：
- 禁止任何"仿佛/似乎/宛若/好像/犹如/如同"开头的句子
- 禁止段落结尾升华（"他明白了……""那一刻……"）
- 禁止用"这不是A这是B"句式
- 禁止用"XX说道"+"XX轻叹"五字块开头
- 禁止用"是的，……""没错，……"做段首
- 不要写"像……一样"比喻句，用动作直接展示
- 不要写角色内心独白超过3句，用行为/对话替代
- 禁止总结性结尾——用动作/场景/对话戛然而止
- Claude版"去AI味"重点：砍比喻，砍排比，砍升华
- 每个自然段最多1个逗号句，用句号断短
```

### Claude特有去AI味优先级

```
① 斩比喻句优先（Claude最爱写"像……一样"）
② 拆长段（Claude倾向写6-8行段落，至少砍一半）
③ 删段尾金句（Claude每个段落都爱落"那一刻……"）
④ 替换优雅动词 → 大白话（"凝视"→"看"，"轻语"→"说"）
⑤ 硬性限制：每1000字不超过3个"仿佛/似乎/好像/犹如"总计
```

### Claude适配写作建议

```
✅ 利用其文学底蕴写「细腻的心理变化」「环境氛围营造」
✅ 适合需要「精美白描」「物象象征」的段落
✅ 写虐心时Claude的细腻是优势——让"轻刺"更刺
❌ 不要让Claude写纯爽文/番茄爆文——容易"文绉绉"
❌ 不要让Claude写系统提示/弹窗——它写出的系统不够"脏"
✅ 写悲剧/BE时Claude是最佳选择——痛感层次感最强
```

## 26.3 DeepSeek 优化分支

### 特性诊断

| 维度 | 表现 | 严重程度 |
|:-----|:-----|:---------|
| 中文水准 | 优秀，自然地道 | ✅ 优点 |
| 文学性 | 中等偏保守 | ⚠️ 中等 |
| 禁用词倾向 | "不禁/不由得/心头一紧"频率高 | 🔴 高 |
| 安全过滤 | 较严格，可能拒稿 | 🔴 注意 |
| 情节推进 | 直接高效 | ✅ 优点 |

### 负面提示专属

```
DeepSeek专用深层负面提示（写章节前注入）：
- 禁止"不禁""不由得""忍不住""下意识地"等被动式开头
- 禁止"心头一紧""内心一沉""心底泛起一丝"模板
- 禁止"他知道""她明白""他意识到"——用行为展示
- 禁止"这件事让他明白了……"总结句式
- 禁止"是的""没错"做段首肯定
- 对话比例强制≥50%，旁白要压缩到极简
- 写冲突时不要"理性分析"——直接写动作和对话
- DeepSeek版"去AI味"重点：砍心理描写，加对话密度
- 负面情绪可用具体行为展示（砸/摔/踹），避免直接说"愤怒"
```

### DeepSeek特有去AI味优先级

```
① 心理描写外化优先（DeepSeek最爱写"他感到……"）
② 删"不禁/不由得/忍不住/下意识地"系列
③ 砍"他知道/她明白/他意识到"——示现不告诉
④ 增加对话密度（DeepSeek天然对话不够多）
⑤ 加小动作/习惯性肢体语言（DeepSeek角色动作偏少）
```

### DeepSeek适配写作建议

```
✅ DeepSeek写「对话交锋」「冲突场景」最自然
✅ 非常适合番茄爽文/系统流——节奏天然快
✅ 适合写「快节奏网文」——不太文艺，直奔主题
✅ 写「规则怪谈」时DeepSeek的冷感是优势
❌ DeepSeek写「细腻情感」「代际伤痛」需要额外引导
❌ DeepSeek容易写"端着"——需要明确要求"口语化"
✅ DeepSeek写「种田文/日常」自然感最强
```

## 26.4 GPT-4 优化分支

### 特性诊断

| 维度 | 表现 | 严重程度 |
|:-----|:-----|:---------|
| 中文水准 | 翻译腔重 | 🔴 严重 |
| 文学性 | 模板化 | 🔴 高 |
| 禁用词倾向 | "同时/此外/然而/因此"连词多 | 🔴 严重 |
| 结构控制 | 极好 | ✅ 优点 |
| 多语言混合 | 易混入英文语序 | 🔴 严重 |

### 负面提示专属

```
GPT专用深层负面提示（写章节前注入）：
- 严格禁止翻译腔句式（"存在着""被……所……""……的是"）
- 严格禁止"同时/此外/另外/值得一提的是"过渡词
- 严格禁止"然而/但是/不过/尽管如此/即使如此"转折词出现在段首
- 严格禁止英文语序定中结构（"那盏挂在墙上的灯"→"墙上那盏灯"）
- 禁止"被"字句（"他被吓了一跳"→"他吓得一哆嗦"）
- 禁止"在……中/里"作状语句式（"在黑暗中，他……"→直接写）
- 禁止"是……的"强调句式（"是他做的"→"他干的"）
- 每个"的"字使用前自问：能删吗？
- 对话要用中文口头习惯——不要写"不，那不是我要的"这种翻译腔
```

### GPT特有去AI味优先级

```
① 翻译腔是大头——优先砍"在……中""被……""……的"
② 砍过渡连词——"同时/此外/然而/因此"全删
③ 改英文语序（"他戴着一顶红色的帽子"→"他戴红帽子"）
④ 砍"是……的"强调句
⑤ 段落变短——GPT喜欢写"起承转合"式段落
⑥ "的"字密度控制——每段不超过3个"的"
```

### GPT适配写作建议

```
✅ GPT的强项是「情节规划」「大纲设计」「结构梳理」
✅ 让GPT写「伏笔设计」「因果链分析」最擅长
✅ 适合写「悬疑推理」的逻辑构建部分
❌ 不要让GPT写纯中文大白话章节——翻译腔需要额外花功夫去
✅ 用GPT时配合anti-ai-polish里的翻译腔专项检测
✅ GPT写「系统流/数据流/弹窗」的格式感最好
```

## 26.5 Qwen/Qwen2.5 优化分支

### 特性诊断

| 维度 | 表现 | 严重程度 |
|:-----|:-----|:---------|
| 中文水准 | 地道，本土化好 | ✅ 优点 |
| 文学性 | 中上 | ⚠️ 中等 |
| 禁用词倾向 | "忽然/突然/瞬间"频率高 | 🔴 高 |
| 结构控制 | 偏保守 | ⚠️ 注意 |
| 创新性 | 依赖模板，容易套路 | ⚠️ 中等 |

### 负面提示专属

```
Qwen专用深层负面提示（写章节前注入）：
- 禁止"突然/忽然/瞬间"作为场景转换工具——用"已完成的事"暗示
- 禁止"他/她/它"开头连续3段——变化主语
- 禁止"这时/就在这时/此刻/此时"时间状语开头
- 禁止"没想到/出乎意料的是/令人惊讶的是"作者评论
- 禁止"在接下来的日子里/在接下来的几天里"时间跳跃
- 禁止"虽然……但是……"句式连续使用
- 单段不超过4行
- 对话中不要用"道："——直接用冒号或无标签
- Qwen版"去AI味"重点：砍时间词，砍模板式过渡
```

### Qwen特有去AI味优先级

```
① 砍时间词（突然/忽然/瞬间/这时/此刻）
② 砍模板句式（"虽然……但是……""不仅……而且……"）
③ 换主语（Qwen倾向同一主语连续多段）
④ 增加口语词汇（"靠""嘶""卧槽""行吧"）
⑤ 增加场景动作切换频率
```

### Qwen适配写作建议

```
✅ Qwen写「日常/种田/都市文」的接地气感最好
✅ 写「现实主义/平庸书写」时本土化最自然
✅ Qwen写「对话」比Claude更"像人说话"
✅ 写「中国背景」相关时文化细节最准确
❌ Qwen写「西方奇幻/翻译文学」不太行
❌ Qwen写「宏大叙事/史诗感」容易套路
✅ 写「爽文打脸」时节奏感合适
```

| 超长上下文 | 200万字，最强 | ✅ 超大优势 |
| 文学性 | 中下 | ⚠️ 注意 |
| 禁用词倾向 | 重复/啰嗦 | ⚠️ 中等 |
| 人物一致性 | 依赖记忆，容易漂移 | 🔴 注意 |

### 负面提示专属

```
Kimi专用深层负面提示（写章节前注入）：
- 禁止重复表达（Kimi喜欢同一意思换说法说两遍）
- 禁止啰嗦描写——每句话必须有推进作用
- 禁止"值得一提的是""不得不说的是"废话
- 禁止同一信息在不同段落重复出现
- 字数控制在2000字以内——Kimi过长容易松散
- 对话不要写"他说""她说"超过3轮——用动作/场景区分
```

### Kimi适配写作建议

```
✅ Kimi的核心优势是「超长上下文」——追连载/大长篇最佳
✅ 让Kimi负责「伏笔管理」「连环因果」——上下文吃得住
✅ 写超长连载时Kimi是首选——不需要截断重喂
✅ Kimi适合写「群像剧」——能记住多角色状态
❌ Kimi不适合写短篇/精致文学——文字质感不够
❌ Kimi写对话容易啰嗦——需要严格控制字数
```

## 26.8 通用优化（适用于所有模型）

无论模型如何，写作前强制注入：

```
### 通用写前防火墙
1. 我不是在写AI范文，我是在写"人写的网文"
2. 读者不会为"优美的句子"付费，只会为"想看的剧情"付费
3. 每多用一个"仿佛"，就离读者远一步
4. 角色对话不说书面语——想想你朋友怎么说话
5. "写完了，还不错"=废稿。"写完了，有哪里不对劲"=可能能看
```

### 跨模型去AI味触发器速查

```
┌──────────────┬──────────────────┬──────────────────────────┐
│ 模型         │ 最严重的AI味      │ 一句话拯救方法            │
├──────────────┼──────────────────┼──────────────────────────┤
│ Claude       │ 比喻/升华/说教    │ "砍比喻，删升华，闭嘴"     │
│ DeepSeek     │ 心理描写/被动式   │ "外化心理，加对话，加速"   │
│ GPT-4        │ 翻译腔/连词      │ "删'的'，砍'同时'，别翻译" │
│ Qwen         │ 时间词/模板式    │ "删'突然'，换主语"         │
│ Kimi         │ 啰嗦/重复        │ "砍一半字数，别重复"       │
│ 豆包         │ 空洞描写         │ "加动作，加对话，加冲突"   │
└──────────────┴──────────────────┴──────────────────────────┘
```

## 26.9 模型感知写作示例

### 示例1：写同一段"他走进一家面馆"

**Claude版**（需深度去AI味）：
```
他推开面馆的门，热气和香气扑面而来，像一只温暖的手抚摸着他的脸。
→ 优化后：他推开门，热气糊了一脸。
```

**DeepSeek版**（需加对话加动作）：
```
他走进面馆，闻到了一股香味，心想今天终于能吃顿热乎的了。
→ 优化后：他推门进去。"老板，一碗面，加辣。"
```

**GPT版**（需砍翻译腔）：
```
在忙碌了一整天之后，他走进了一家位于街角的小面馆。
→ 优化后：他拐进街角那家面馆。一天没吃饭了。
```

**Qwen版**（需删时间词）：
```
突然，他感到肚子饿了，于是走进了一家面馆。
→ 优化后：他饿了。推开面馆的门就是喊："老板，大碗。"
```

> **核心原则**：没有"最好的模型"，只有"最适合特定题材+经过去AI味处理的模型"。小说技能的核心不是让AI"写得好"，而是让AI"写得像人"。
