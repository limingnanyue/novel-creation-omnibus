# 修改工作流 · Revision Workflow

## 适用场景
- 初稿完成后的修改
- 结构性重写（大改）
- 逐句精修（小改）
- 读者反馈整合

---

## 1. 修改的四个层次

| 层次 | 改什么 | 什么时候做 | 耗时比例 |
|:-----|:-------|:----------|:---------|
| 🏗️ 结构修改 | 大纲/情节/节奏/POV/章节顺序 | 第一次修改 | 40% |
| 📐 章节修改 | 每章的起承转合/信息量/密度 | 结构确定后 | 25% |
| ✏️ 逐句精修 | 句子流畅度/用词/节奏 | 章节确定后 | 25% |
| 🔍 最终校对 | 错别字/标点/格式 | 发布前 | 10% |

---

## 2. 结构修改

### 2.1 结构自检清单
- [ ] 故事的起承转合是否完整？
- [ ] 章节顺序是否最佳？（有没有可以调换的？）
- [ ] 节奏是否有长篇平坦区？（连续5章无事发生）
- [ ] 每条线是否都有推进？
- [ ] 角色的动机是否前后一致？
- [ ] 有没有可以合并的章节？
- [ ] 有没有需要拆分的章节？
- [ ] 信息释放的顺序是否合理？
- [ ] flashback的位置是否合适？
- [ ] 高潮是否在最佳位置？

### 2.2 结构修改的"打乱重排"法
1. **提取每章的核心事件**（1句话概括）
2. **打乱所有卡片**
3. **重新排列**（新的可能比原来的好）
4. **去掉没有推动力的章节**（删了也不影响故事的）
5. **标记需要新增的连接章节**

### 2.3 添加删减的判断标准
| 判断标准 | 保留 | 删除 |
|:---------|:-----|:-----|
| 推动情节 | ✓ 保留 | — |
| 塑造人物 | ✓ 保留（但可浓缩） | — |
| 建立氛围 | ✓ 保留（但不过量） | — |
| 提供信息 | ✓ 保留（但换位置） | — |
| 以上都不符合 | — | ✗ 删除或合并 |

---

## 3. 章节修改

### 3.1 每章的三段式检视
```
开头 → 前3段是否抓住读者？
中段 → 信息量是否适中？节奏是否恰当？
结尾 → 是否有章末钩子或情感收束？
```

### 3.2 章节修改的"五问法"
1. **这一章的核心是什么？**（1句话能说清吗？）
2. **读者读完这一章的情绪是什么？**（兴奋/感动/紧张/无聊）
3. **有没有可以删掉的段落？**（对核心无用的=删）
4. **有没有可以浓缩的段落？**（太啰嗦的=剪）
5. **结尾是否让读者想读下一章？**（不是=重写结尾）

### 3.3 对话修改的专项检查
- [ ] 每个角色的语气是否一致？
- [ ] 对话是否有"信息炸弹"（太明显的信息输送）？
- [ ] 对话中是否有可以转化为潜台词的部分？
- [ ] 对话前后的动作/神态是否缺失？
- [ ] 对话是否过长需要穿插？

## 4. 逐句精修

### 4.1 句子层面的"七刀"
| 刀数 | 剪什么 | 修改方向 |
|:-----|:-------|:---------|
| 1 | "XX地/XX的" | 替换为具体动作 |
| 2 | 被动语态 | 改主动语态 |
| 3 | "很/非常/特别" | 删除后用具体描述 |
| 4 | "开始/继续/持续" | 直接写动作 |
| 5 | "感到/觉得/意识到" | 直接用感受呈现 |
| 6 | "仿佛/似乎/好像" | 删除2/3 |
| 7 | "是……的"句式 | 简化 |

### 4.2 用词多样化的三个原则
1. **不为换而换** — 同一个词隔两三句再用没问题
2. **语境优先** — 用最准确的词，不是最复杂的词
3. **角色对话用词要匹配人设** — 大学教授和街头混混用词不同

### 4.3 节奏的句子级调整
```
短句（<10字）：加速、制造紧张
中句（10-20字）：标准叙事节奏
长句（20+字）：放缓、制造沉浸感
```
- 连续3个短句 → 读者紧张
- 连续3个长句 → 读者疲劳
- 短长交替 → 最舒适的阅读体验

## 5. 读者反馈整合

### 5.1 反馈分级处理
| 反馈类型 | 优先级 | 处理方式 |
|:---------|:-------|:---------|
| 逻辑bug | 🔴 立即 | 无条件修正 |
| 角色行为矛盾 | 🔴 立即 | 检查人设是否一致 |
| 节奏问题（太慢） | 🟡 高 | 缩短相关章节 |
| 情感不够 | 🟡 高 | 增加情感描写/内心戏 |
| 设定看不懂 | 🟡 中 | 简写设定/加注/信息前置 |
| 个人口味问题 | 🟢 低 | 考虑但不必须 |
| "我觉得应该写成X" | ⚪ 参考 | 感谢但不采用 |

### 5.2 修改的"度"
- **不要被读者牵着走** — 每个读者都是编辑，但不是每个都对
- **多数人反应 = 问题** — 如果5个人说同一问题，必须改
- **少数人的强烈建议 = 有道理但未必适合** — 考虑核心方向是否一致

## 6. AI味检测·修改相关

### 检查清单
- [ ] 是否跳过了结构修改直接进入精修？
- [ ] 每个章节是否能1句话说清核心？
- [ ] 结尾有钩子吗？没有→重写
- [ ] 是否删除了5%以上不必要的字数？
- [ ] 对话是否有"信息炸弹"？
- [ ] 长句短句是否交替使用？
- [ ] 是否有读者指出过的同一问题未改？
- [ ] 不同层次的修改是否按顺序而非跳跃进行？
