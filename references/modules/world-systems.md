# 世界运转体系 · World Systems

## 适用场景
- 力量等级/修炼体系设计
- 经济系统/货币体系
- 政治格局/阵营/势力
- 组织架构/帮派/公司
- 社会阶层/权力结构
- 历史背景/编年史

---

## 1. 力量体系设计

### 1.1 力量等级设计三原则
1. **可感知的差距** — 高一级就是打不过，让读者明确知道谁强谁弱
2. **升级要有代价** — 资源/时间/心魔/副作用（否则变数值游戏）
3. **不可无限膨胀** — 设定终极天花板，否则后期力量崩塌

### 1.2 等级命名与差异化
| 等级数 | 推荐命名方式 | 示例 |
|:-------|:------------|:-----|
| 3级 | 入门→精通→大师 | 简单明了 |
| 5级 | 初→中→上→顶→极 | 中等复杂度 |
| 7级 | 练气→筑基→金丹→元婴→化神→大乘→渡劫 | 修仙标配 |
| 9级 | 1-9阶/一品到九品 | 数值化好记 |
| 12+级 | 需要每3-5个等级做一个大境界区分 | 避免混乱 |

### 1.3 跨级战斗的合理范围
| 等级差 | 战斗结果 | 说明 |
|:-------|:---------|:-----|
| 同级 | 五五开，看装备/经验/环境 | 标准对决 |
| 高1级 | 七三开，低才有机会 | 常规战斗 |
| 高2级 | 九一开，除非天时地利 | 难度挑战 |
| 高3级 | 碾压，低方只能逃/被救 | 必败/经典装逼打脸场景 |

**注意：** 跨越3级以上越级反杀→必须有大代价（神器/燃寿/舍命）

### 1.4 力量体系的延展性
好的力量体系要预留"扩展接口"：
- 种族差异（兽人有力量加成、精灵有敏捷加成）
- 属性克制（火克木、光克暗）
- 体系冲突（修仙 vs 魔法 vs 科技）
- 残缺/特殊血脉/变异

## 2. 经济系统

### 2.1 货币与资源体系
| 资源层级 | 获取难度 | 流通性 | 设计要点 |
|:---------|:---------|:-------|:---------|
| 基础货币 | 易得 | 强 | 日常交易用，不能断供 |
| 中级资源 | 中 | 中 | 修炼/打造装备用，约束升级速度 |
| 顶级资源 | 极难 | 弱（各势力垄断） | 引发争端的导火线 |

### 2.2 经济的叙事功能
```
经济不是数字游戏——它是推动情节的工具。
```
| 功能 | 用法 | 示例 |
|:-----|:-----|:------|
| 制造困境 | 主角缺钱/负债 | 修仙主角买不起丹药→必须冒险 |
| 制造冲突 | 资源争夺 | 两个家族争夺矿脉→爆发战争 |
| 展示格局 | 贫富差距 | 底层平民吃糠咽菜→贵族挥金如土 |
| 施压/通牒 | 经济封锁/惩罚 | 商会断了主角的供货→被迫寻找新路 |

### 2.3 物价的"真实感"锚定
给读者一个熟悉的参照物：
```
一个包子=1铜钱 → 一碗面=5铜钱 → 住店一晚=20铜钱
→ 一本普通功法=500金币（=50000铜钱）
→ 一把神兵=5000金币（=500000铜钱）← 够一家人吃500年
```

## 3. 政治格局/势力

### 3.1 势力关系的最简模型
```
至少三个势力，构成动态平衡：
- 敌对的（正面对抗）
- 中立的（可拉拢/可背叛）
- 潜在的（台面下的威胁）
```

### 3.2 势力关系变化的六种模式
| 模式 | 描述 | 叙事价值 |
|:-----|:-----|:---------|
| 结盟 | 两个势力联合 | 形成压力 |
| 背叛 | 盟友反目 | 剧情转折 |
| 兼并 | 大鱼吃小鱼 | 格局改变 |
| 渗透 | 安插内鬼/间谍 | 暗线丰富 |
| 平衡 | 三足鼎立 | 稳定的冲突 |
| 崩塌 | 势力瓦解 | 权力真空/新秩序 |

### 3.3 势力设计的记忆法则
```
每个势力最多给读者3个记忆点：
1个价值观（门派精神/家训）
1个标志性人物（掌门/族长）
1个特色制度（考核/奖惩/入会）
```

## 4. 组织/制度设计

### 4.1 常见组织类型模板
| 组织类型 | 典型结构 | 冲突来源 |
|:---------|:---------|:---------|
| 宗门/门派 | 掌门→长老→真传→内门→外门 | 资源分配/传承之争 |
| 家族/世家 | 族长→嫡系→旁系→家臣/客卿 | 继承权/家族荣誉 |
| 商会/公会 | 会长→理事→会员→学徒 | 利益/竞争/背叛 |
| 军队/势力 | 将军→副将→百夫长→士兵 | 忠诚/战略/政变 |
| 学院/学府 | 院长→教授→优等→普通 | 流派之争/天才之争 |

### 4.2 制度的"不完美"设计
好的制度设计要有漏洞——制造冲突：
- 考核制度有漏洞→有人作弊/走后门
- 继承规则有歧义→引发内斗
- 晋升条件太苛刻→天才被埋没
- 惩罚制度不公→主角反抗

## 5. 社会阶层

### 5.1 社会阶层的"流动"设计
| 流动类型 | 描述 | 叙事潜力 |
|:---------|:-----|:---------|
| 向上流动 | 底层→顶层 | 爽文主线（打脸/逆袭） |
| 向下流动 | 顶层→底层 | 重生/跌落神坛 |
| 平行流动 | 一个领域→另一个领域 | 跨界/新地图 |
| 伪装流动 | 隐藏身份/装弱 | 扮猪吃虎 |

### 5.2 阶层差异的微观体现
```
高阶层和低阶层在同一个场景里的不同体验：
同一条街——贵族坐轿经过去茶楼，乞丐在墙角缩着
同一间客栈——上房有独立院子，通铺挤了十个人
同一场比试——有丹药的vs没有丹药的
```

## 6. 历史与编年史

### 6.1 编年史的"锚点事件"
只设计5-8个锚点事件，形成历史骨架：
```
史前→千年战争（古神陨落）
纪元1年→第一王朝建立
纪元500年→大灾变（导致力量体系变化）
纪元1000年→第二次大战（当前格局形成）
纪元1500年→"现在"的时间起点
```

### 6.2 历史的叙事功能
- **背景驱动**：现在的冲突源自历史恩怨
- **伏笔来源**：历史上的秘密/遗迹等待被发现
- **规则解释**：为什么这个世界是这样的？
- **人物动机**：角色的家族/国度历史塑造他们的目标

### 6.3 "冰山原则"用于历史
```
只露出10%的历史给读者看
剩下90%你自己知道就好
但露出的10%必须让读者感觉到下面还有冰山
```
