## 13.1 同人创作总流程

```
Phase 0：检测是否含题材标签，加载对应引擎
Phase 1：意图解析+模块调度
Phase 2：五步查证+角色档案+知识图谱
Phase 3：OOC预检+世界观一致性+读者模拟+题材校准
Phase 4：实时查证+OOC自检+一致性检测+考据融入
Phase 5：全文OOC评分+一致性扫描+读者评估
Phase 6：AU声明+内容预警+来源引用
```

## 13.2 OOC检测量化评分

```
总分 = Σ(偏离度×权重) × 修正系数

权重：核心性格30% / 能力边界25% / 关系逻辑20% / 语言风格15% / 价值观10%
修正系数 = AU声明(0.3~1.0) × 原作留白(0.8~1.0) × 读者预期(0.7~1.0)

0-15🟢安全 / 16-35🟡低风险 / 36-55🟠警告 / 56-75🔴危险 / 76-100⚫致命

四问法：
问1：原作TA会这样做吗？→会→继续 / 不会→问2
问2：在AU范围内吗？→是→标注 / 否→问3
问3：有原作线索支撑吗？→有→标注合理推断 / 无→必须修改
问4：破坏世界观一致性吗？→检测
```

## 13.3 历史考据四步

```
Step 1 确定范围：朝代+维度
Step 2 执行查证：优先中华经典古籍库/中华古籍资源库
Step 3 整理结果：记录出处，区分正史与推测
Step 4 融入创作：关键细节标注来源

S级渠道：read.nlc.cn(国图) / jingdian.ancientbooks.cn(中华书局)
A级渠道：shidianguji.com / new.shuge.org / ctext.org

食材传入：玉米(明)/番薯(明)/土豆(明末)/辣椒(明末)/菠菜(唐)/西瓜(五代宋)
称谓规范：爸爸→父亲/阿爷/阿娘  老公→夫君/娘子  皇上各朝不同
历代官制：秦汉三公九卿 / 隋唐三省六部 / 宋二府三司 / 明内阁六部 / 清军机处
```
