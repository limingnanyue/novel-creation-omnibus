# 第1章：长篇正文写作·完整6步流程

> 适用："写第X章""写下一章""写正文"

---

## Step 1：读取最新状态

写正文前必须读取以下文件，缺一不可：

```
□ 人物库 → 确认出场角色及当前状态
□ 世界观 → 确认世界规则与设定
□ 全书大纲 → 确认当前章节在整体结构中的定位
□ 状态追踪 → 角色实时位置/伤势/物品/关系
□ 本章细纲文件 → 确认本章剧情规划与情节点
□ 记忆台账 → 确认上章结尾类型（避免重复）、伏笔状态
□ 章节摘要 → 回顾整体故事进程
□ 上章最后2段 → 确保直接承接，不写时间词
□ 热梗仓库 → 匹配适合场景的热梗
```

## Step 2：写正文（同时遵守以下全部标准）

### 📏 字数标准
- 通用长篇：**2000-2500字**（🚨 低于2000必须扩写）
- 开头200字内进入冲突/笑点/悬念
- 每500字一个小转折/小爽点
- 对话占比≥50%
- 全用短句、口语化，无比喻/排比/拟人
- 情绪用动作展示，不直说
- 章末最后3句话=强力钩子

### 🔗 连续性标准
- 章尾留钩子，不闭环收尾
- 章首第一句直接承接上章结尾（场景/动作/对话）
- 🚫 章首禁用任何时间词：第二天、次日、翌日、隔天、早上、下午、晚上、清晨、一小时后、半天后、片刻之后、就在这时
- 🚫 禁用隐性过渡短语：转眼之间、过了一会儿、不知过了多久
- ✅ 用"已完成的事"暗示时间流动："掐掉视频的时候天已经亮了"
- 🚫 连续2章不能使用同一类型的结尾模式
- 🚫 连续2章不能都用"隔夜/隔天"开头
- 📈 每章至少包含2-3个剧情要素：冲突/发现/抉择/反转/关系变化/伏笔

### 🔧 DS优化标准
- 第一段从身体感受开始，不从认知开始
- 角色遇到危机：身体反应(0-3秒)→情绪(3-10秒)→理智(10秒+)
- 句号占比≤30%，连续句号不超过3个
- 段落长短交替：长(100-200)+中(50-100)+短(10-50)+极短(1-10)
- 对话有潜台词，非一问一答
- 环境描写覆盖≥3种感官（视觉+听觉+触觉+嗅觉+味觉）
- 一个场景内只允许一个视角

### 🗑️ 反水文标准
- 无"想了想""叹了口气""愣了一下"等废话填充词
- 无同一信息说两遍
- 无"没有A，没有B，也没有C"否定式排比
- 无无意义动作描写
- 对话不超过3轮无新信息

## Step 3：优化（三步必走）

```
第一步：语法优化
  - 检查句号滥用（改逗号/破折号）
  - 检查主语冗余（删重复主语）
  - 检查重复表达（换说法或合并）
  - 检查AI口癖（突然/瞬间/不禁/不由得）

第二步：去AI味
  - 六大门禁（见第8章§8.1）
  - 三刀流（见第16章§16.2）
  - 八类必查（见第8章§8.3）

第三步：连续性验证
  - 优化后章首是否被插入了时间词？
  - 章尾钩子是否被误删了？
  - 承接是否仍连贯？
```

## Step 4：同步文档（9项全更新）

```
□ 章节摘要 → 追加本章摘要（2-3句话概括）
□ 人物库 → 更新出场角色记录+好感度变化+状态变化
□ 状态追踪 → 更新位置/积分/等级/系统状态/伤势
□ 情绪曲线 → 追加本章六维情绪数据（爽/甜/虐/燃/悬/暖）
□ 钩子密度 → 追加本章钩子类型+强度评级
□ 细节注册表 → 更新惯用动作/口头禅/场景锚点
□ 记忆台账 → 追加章节记录+更新结尾类型汇总+更新伏笔状态
□ 对话记忆 → 追加本轮对话精简摘要
□ 热梗仓库 → 检查是否超6小时未更新，是则更新
```

## Step 5：每5章更新全书大纲

```
□ 更新总览表格（章节数/字数/完成比例）
□ 追加新完成阶段的剧情对照表
□ 更新后续章节规划
□ 更新伏笔状态与写作铁律
```

## Step 6：每10章运行因果链分析

```
□ 构建因果链（起因→逐级因果→结果）
□ 人物信息差分析（知道/不知道/想误导/想隐藏）
□ 章节功能定位
□ 内容成分比例分析
□ 保存报告
```
