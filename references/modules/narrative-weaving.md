# 多线叙事 · Narrative Weaving

## 适用场景
- 多视角/POV切换写作
- 多线剧情并行推进
- 时间线交错管理
- 线索铺设与回收

---

## 1. POV与视角管理

### 1.1 三种主要视角
| 视角 | 优势 | 限制 | 适用题材 |
|:-----|:-----|:-----|:---------|
| 第一人称"我" | 代入感最强、心理活动直接 | 不能知道别人想法、信息受限 | 悬疑/言情/都市 |
| 第三人称有限（一个角色） | 近距离+一定程度的信息控制 | 只能写该角色感知到的事 | 绝大多数网文 |
| 第三人称全知 | 全景展示、多条线同时推进 | 代入感弱、容易"上帝视角" | 史诗/群像/战争 |

### 1.2 POV切换规则
**黄金法则：同一场景只用一个POV。**

| 切换维度 | 规则 | 示例 |
|:---------|:-----|:------|
| 章节级切换 | 每章1个POV，下一章换 | 第3章（男主POV）→ 第4章（女主POV） |
| 段落级切换 | 必须有空行/分隔线，一次切换 | "与此同时，在城市的另一端……" |
| 禁止 | 同一段落/同一对话中切换 | ❌ "他笑着说。而她心里想的是另一回事。" |

### 1.3 POV切换信号
切换时给读者明确的信号，不要让读者猜：
- **空间信号**："三公里外的指挥室里……"
- **时间信号**："就在同一时刻……"
- **人物信号**："让我们看看另一边发生了什么……"

## 2. 多线叙事结构

### 2.1 三条基本线的分工
| 线 | 功能 | 节奏 | 占章比例 |
|:---|:-----|:-----|:---------|
| **A线（主线）** | 推动核心冲突 | 最快，有进展驱动 | 60-70% |
| **B线（副线）** | 丰富人物/世界观/感情发展 | 中等，有时停滞 | 20-30% |
| **C线（暗线）** | 伏笔/真相/隐藏冲突 | 最慢，偶尔露头 | 5-15% |

### 2.2 主流叙事结构

#### 平行式（不断切换的视角）
```
A1 → B1 → A2 → C1 → A3 → B2 → A4 → C2 → ...
```
适用：悬疑/冒险/双主角

#### 汇流式（多条线渐渐交汇）
```
A1 → A2 → B1 → A3 → B2 → C1 → A4 → A&B交汇 → A&B&C交汇
```
适用：群像/史诗/群穿

#### 回溯式（通过回忆闪回交织）
```
现在线A → 闪回B1 → 现在线A2 → 闪回B2 → 现在线A3 → 交汇点
```
适用：悬疑/虐文/深度心理

### 2.3 多线交织的节奏控制
```
主线——██████████████████████████████████████
副线——   ▓▓▓▓▓▓▓▓▓          ▓▓▓▓▓▓▓
暗线——       ░░░               ░░░░      ░░░░
```
- 主线不间断推进
- 副线每5-8章出现一次
- 暗线每15-20章露头一次，每次多一些信息

## 3. 时间线管理

### 3.1 时间锚定
每次时间跳跃（闪回/快进/平行时间）必须提供**时间锚点**：
```
"三天前……"
"那是半个小时以前的事了……"
"同样在这个深夜，千里之外的……"
```

### 3.2 闪回使用的三条铁律
1. **闪回不可修复糟糕的当下叙事** — 如果当前情节无聊，加闪回只会更糟
2. **闪回必须携带当前叙事需要的信息** — 每次闪回回答一个"为什么"
3. **闪回位置要算好** — 放在小高潮之后，不要在说高潮前打断

### 3.3 时间旅行/重生时间线管理
| 类型 | 管理策略 | 注意 |
|:-----|:---------|:-----|
| 重生回到过去 | 已知未来的"先知"时间线 | 避免蝴蝶效应混乱 |
| 时间循环 | 每次循环递增的信息量 | 读者能忍3-5次循环 |
| 平行时间线 | 每条线独立的因果 | 必须标记清楚 |

## 4. 线索铺设与回收

### 4.1 线索的四种类别
| 类别 | 最大长度 | 举例 |
|:-----|:---------|:-----|
| 章节级线索 | 3-5章内回收 | 某人消失的钥匙→其实掉在车里 |
| 卷级线索 | 1卷内回收（30-50章） | 神秘人的身份 |
| 全书级线索 | 大结局回收 | 最终boss的身份/世界真相 |
| 系列级线索 | 跨书/下一部 | 多本书共享的秘密组织 |

### 4.2 线索密度公式
```
每10章 = 2-3个新线索投放 + 1条旧线索回收
```
- 太多线索→读者记不住（信息过载）
- 只投不收→读者失望（悬空感）
- 回收后没新线→情节停滞（推进无力）

### 4.3 线索铺设技巧
| 技巧 | 做法 | 示例 |
|:-----|:-----|:------|
| 轻描淡写 | 看似不经意地提到 | "走廊尽头那扇门从来没人打开过" |
| 重复强调 | 不同场景提到同一个细节 | 三次提到"路边的栀子花开了" |
| 反常识 | 细节违反常理 | "大夏天他却裹着厚围巾" |
| 信息差 | 不同角色知道不同的事 | A知道钥匙在哪/B不知道/C在找 |

### 4.4 线索回收时机
| 线索类型 | 回收前最大埋藏长度 | 读者反应 |
|:---------|:------------------|:---------|
| 短线索 | 3-5章 | "哦原来是这样！" |
| 中线索 | 10-20章 | "靠，前面就暗示过！" |
| 长线索 | 30-50章 | "我回头重看了一下，真在第X章！" |

## 5. AI味检测·多线叙事

### 检查清单
- [ ] 同一场景是否只有1个POV？
- [ ] POV切换时是否有明确信号？
- [ ] 闪回是否有时间锚点？
- [ ] 每条线是否都有自己的"进展"？
- [ ] 有没有超过10章没有进展的废线？
- [ ] 最近5章有没有回收至少1条线索？
- [ ] 线索投放是否不超过3条/10章？
- [ ] 有没有已投放超过30章还没回收的长线索？
- [ ] 主要角色是否有清晰的视角标签？
- [ ] 时间跳跃是否有行文上的视觉区分（分隔线/空行/章节换）？
